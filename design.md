# mindlens 디자인 시스템 — Furiosa 차용 (정본)

> **정본 규칙:** 이 문서가 디자인 시스템 SoT다. 토큰 값을 고칠 땐 여기서 고치고 `web/styles/globals.css`의 `@theme`·`tailwind.config.ts`를 **같이** 갱신한다(둘은 이 문서의 구현). 구현 중 "느낌"으로 색·간격을 바꾸지 말 것 — 먼저 이 문서를 고친다. (하네스 §2·§4)

>

## 0. 큰 원칙 — 이원화 (도구 vs 대화)

mindlens는 청중이 둘이라 **한 시스템, 두 표면**으로 간다. 색이 화면의 성격을 가른다.

| 표면 | 대상 | 스킨 | 그라운드 | 라디우스 | 인상 |
|---|---|---|---|---|---|
| **의뢰자** (대시보드·가이드·결과) | 리서처(도구 사용) | **Furiosa 정통** | 화이트/페이퍼 | 8–10px (타이트) | 데이터·신뢰·하드웨어 시그널 |
| **응답자** (동의·스크리너·인터뷰·완료) | 인터뷰 참가자(대화) | **Furiosa 웜** | 크림/블러시 | 12–16px (부드럽게) | 사람·따뜻함·안심 |

- **레드는 두 표면 공통 액센트**지만, 도구 쪽은 절제된 액션·데이터 강조로, 대화 쪽은 따뜻한 액션으로 쓴다.
- **블랙 드라마(풀블리드 검정 히어로) 배제** — obsidian은 텍스트·아이콘·소형 요소에만. 그라운드는 항상 라이트.
- 랜딩/마케팅은 의뢰자 톤(라이트·레드)을 따르되 카피는 따뜻하게.

## 2. 타이포그래피

- **서체: Pretendard**(한글+라틴 단일 패밀리). ABC Favorit(Furiosa 독점·한글 없음) 대체. 웹폰트는 `globals.css`에서 로컬/셀프호스팅.
- **mono: 시스템 monospace**(`ui-monospace, "SF Mono", Menlo, Consolas, monospace`) — **오직 텔레메트리 수치(파이프라인 진행 지표 등)에만.** 그 외 mono 역할(메타·라벨)은 Pretendard + `tabular-nums`.
- **시그니처: 크고 얇은(400) 타이트한 제목** + 본문 500.

| 역할 | 크기 | 굵기 | 자간 | 비고 |
|---|---|---|---|---|
| display | clamp(32→54px) | 400 | -0.03em | 히어로. 얇고 타이트 |
| title | 24–28px | 600 | -0.02em | 화면 제목 |
| section | 18–19px | 600 | -0.01em | 섹션 제목 |
| body | 15px | 500 | 0 | 본문 |
| meta | 12–13px | 500 | 0 | Pretendard + `tabular-nums` |
| eyebrow | 11–12px | 500–600 | +0.1em, UPPER | mono 렌더(라벨) |
| telemetry | 12–13px | 400 | 0 | **시스템 mono**(텔레메트리 수치 전용) |

숫자 정렬 필요한 곳 어디나 `font-variant-numeric: tabular-nums`.

## 3. 형태 · 깊이

- **라디우스:** 버튼/입력 8px · 의뢰자 카드 10px(토큰 `rounded-card`, `<Card>` 전용) · 응답자(웜) 카드 12–16px · 배지 6px · pill 9999px(탭/칩).
- **보더:** 1px `silver`/`platinum` 헤어라인이 기본. 무거운 보더·강한 드롭섀도 금지.
- **섀도:** 소프트·저불투명 — `card 0 1px 2px rgba(13,13,13,.04)`, `lift 0 4px 20px rgba(13,13,13,.06)`. 웜 카드는 웜 섀도 `rgba(140,70,20,.06)`.
- **밴드 교대:** canvas ↔ paper 로 섹션 구분(보더 남발 대신).

## 4. 아이콘 (하네스 §2 하드룰)

- **아이콘 = `lucide-react`. 제품 시각 UI에 이모지 금지**(폰트·정렬·접근성·톤).
- 교체 매핑(기존 이모지 → lucide): 🎤→`mic` · 🔊→`volume-2` · ●녹음→`circle`(채움) · 정지→`square` · ✅→`check-circle-2` · 🙏→`heart-handshake` 또는 텍스트 · 🎙 진행자→`mic` · 검색→`search` · 새로만들기→`plus` · 더보기→`more-horizontal` · 프로젝트→`folder` · 에러→`alert-triangle` · 재시도→`rotate-cw`.
- 이모지 교체는 **횡단 변경** → 코드 전 전수조사(스키마·컴포넌트·테스트, 하네스 §2).

## 5. 컴포넌트

### 앱 셸 (의뢰자) — 좌측 사이드바
- 브랜드 마크 + `search`(⌘K) + 네비(**실존 화면만**: 프로젝트).
- **`프로젝트` 네비 아래 최근 프로젝트 서브리스트**(최대 8개, 최근 생성순 — 초과 시 `전체 보기` → `/projects`). 행 = 상태 점(작성 중=red·배포됨=obsidian·종료=grey) + 말줄임 제목, 활성 프로젝트는 네비와 동일하게 `blush` 배경/red 텍스트로 강조. 로딩 중엔 스켈레톤 2~3행, 0개면 옅은 안내문만("아직 프로젝트가 없어요") — **여기 CTA를 다시 넣지 않는다**(아래 원칙과 동일 이유). fetch 실패는 서브리스트만 조용히 숨긴다(사이드바 본체는 안 깨뜨림).
- **`+ 새 프로젝트` CTA 는 사이드바에 두지 않는다** — 각 페이지 헤더에만. 둘 다 두면 첫 화면에 같은 버튼이 두 개 보인다.
- **무인증:** 워크스페이스 전환·계정 메뉴·유저 아바타 **없음.** 사이드바 푸터 = "로그인 없이, 링크만 공유" 안내(무인증을 특징으로).
- 활성 네비 = `blush`/red-wash 배경 + red 아이콘.

### 콘텐츠 헤더바 (의뢰자) — `/projects/*` 공통
- 콘텐츠 컬럼 최상단의 슬림 바(52–56px, sticky). 화이트 바탕 + 하단 1px `silver` 보더. 사이드바 두 항목만으론 상세 화면에서 돌아갈 동선이 없던 문제(원래 있던 BackLink가 사이드바 셸 도입 때 빠짐)를 메운다.
- 뒤로 화살표(`arrow-left`, ≥44px 탭 타깃, `aria-label="뒤로"`) + 브레드크럼(obsidian/charcoal 텍스트, `grey` `/` 구분자).
- 경로별: `/projects`="프로젝트"(뒤로 없음) · `/projects/new`="← 프로젝트 / 새 프로젝트" · `/projects/[id]`="← 프로젝트 / {제목}"(제목은 fetch — 로딩 중 스켈레톤, 실패 시 원문 id 대신 "프로젝트"로 대체).
- 상세 화면 안의 탭(인터뷰 가이드/결과)은 그대로 둔다 — 이 바는 탭을 대체하지 않고 그 위 계층(어느 프로젝트인지·어떻게 돌아가는지)만 담당한다.

### 버튼
- **primary:** `brand-red` 배경/흰 글자, 8px. hover `red-dark`.
- **secondary:** 흰 배경/obsidian 글자, 1px `silver`.
- **ghost:** 투명/charcoal.
- (웜 표면에선 동일 구조, 웜 보더/배경.)

### 카드 (프로젝트)
- 흰 배경, 1px `silver`, 10px. 상태 배지(배포됨=obsidian칩·작성중=red-wash칩·종료=grey칩), `more-horizontal`, 하단 익명 카운트(응답 N·진행중 M) + 완료율.
- **응답자 아바타 스택 금지**(익명) → 카운트/진행률 시각화로.

### 상태 화면 (데모 티 제거 — "불러오는 중…" 텍스트 폐기)
- **로딩:** 스켈레톤 시머(텍스트 없음).
- **빈:** lucide 아이콘 + 안내문 + 액션(CTA).
- **에러:** `maroon` 톤 + `alert-triangle` + `rotate-cw` 재시도.

### 토스트 (액션 피드백) — 데모 명확성
> 정본 추가(데모 UX 강화). 컴포넌트 = `web/components/shared/toast.tsx`, 마운트 = `app/projects/layout.tsx`.
- **성공 액션은 토스트로 확인한다** — 저장·배포·내보내기·삭제 등. 화면 어디서 눌러도 **같은 자리**(우하단, 모바일 상단)에서 확인되게. 예전엔 페이지 최하단 인라인 텍스트라 스크롤 위치에 따라 놓쳤다.
- **3.5초 자동 소멸**, `role="status" aria-live="polite"`. 진입 200ms / 소멸 150ms(exit가 enter보다 빠르게, §7).
- **성공 = `go`(초록 체크 `check-circle-2`)**, **에러/파괴 = `maroon`(`alert-triangle`)** — brand-red 는 정상 액션 색으로만(§1). 토스트에 red 채움 남발 금지.
- **폼 필드 에러·dirty 경고는 인라인 유지**(필드 근처). 토스트로 옮기는 건 "액션 결과"만.
- 아이콘 lucide, 이모지 금지(§4).

### 확인 다이얼로그 (파괴적 작업)
- **되돌릴 수 없는 작업에만** 쓴다(프로젝트 삭제 등). 브라우저 `confirm()` 금지 — 스타일을 못 입히고 무엇이 함께 사라지는지 담을 수 없다.
- 구성: `alert-triangle`(maroon 원) + 제목 + **무엇이 함께 지워지는지 수량으로** + `되돌릴 수 없어요` + [취소][삭제].
- **확인 버튼은 `maroon`.** brand red 와 섞지 않는다(§1 시맨틱) — red는 정상 액션 색이라 파괴 액션에 쓰면 손이 먼저 간다.
- **열릴 때 포커스는 취소에.** 엔터 연타로 실행되면 안 된다. `Escape`로 닫힌다(작업 중에는 잠금).
- 트리거 버튼은 `ghost` + `trash-2`, 평상시 `grey` / hover `maroon`. 주 동선에서 비켜 둔다.

### 작업 진행(파이프라인) 화면 (의뢰자)
수 초~수 분 걸리는 서버 작업(가이드 생성·인사이트 생성)이 도는 동안 띄우는 **전체 화면.**
서버가 SSE 로 실제 단계를 흘려보내고 이 화면이 받는다(`api/services/progress.py`).

- **스켈레톤과 섞지 않는다** — 1초 미만 예상이면 스켈레톤, 그 이상이면 이 화면. 둘을 겹쳐 쓰지 않는다.
- **배치:** 콘텐츠 영역 전체. 사이드바·콘텐츠 헤더바는 유지(길을 잃지 않게).
- **구성:** 제목(무엇을 하는 중인지) → mono 경과 타이머 → 단계 리스트 → 하단 요약 바(n/N 단계 · 토큰 · 백그라운드로 두기).
- **단계 아이콘** (전부 `lucide-react`, **이모지 금지** — §4):
  `check`=완료 · `loader-2`(회전)=진행 · `circle`=대기 · `minus`=건너뜀 · `alert-triangle`=실패.
- **색:** 진행 중 `red`(NPU가 일하는 중) · 완료 `go` · 대기·건너뜀 `ink-faint` · 실패 `maroon`(brand red 와 절대 안 섞음 — §1 시맨틱).
- **수치 = 텔레메트리 mono.** 소요 시간·토큰·건수·모델명은 **서버 실측만** 쓴다. 클라이언트가 추정하지 않으며, 값이 없으면 `—`. 상단 경과 타이머만 클라이언트 벽시계다.
- **건너뛴 단계를 완료로 위장하지 않는다.** 자료 없는 프로젝트의 근거 검색처럼 실제로 안 한 일은 `minus`로 표시하고 사유를 적는다.
- **모션:** `loader-2` 회전 + **활성 단계 강조**(현재 `run` 행에 red 좌보더 + 옅은 배경) + 단계가 `done` 될 때 `check` 팝인(scale 0.8→1, 150~200ms). `prefers-reduced-motion` 은 globals.css 전역 규칙이 처리한다.
- **중단 버튼 문구는 "백그라운드로 두기".** 이미 떠난 NPU 호출은 못 죽이므로 "취소"는 거짓말이 된다.

> 정본: `docs/specs/2026-07-24-guide-topics-turn-budget.md` · `docs/specs/2026-07-24-qualitative-narrative-guide.md`
- **이건 설문이 아니라 1:1 심층 인터뷰다.** 질문은 **사건 재구성형**("최근 ~했던 때를 떠올려…어떤 상황이었나요?"). 랭킹·빈도·예/아니오 설문 문형 금지, 같은 선택지 격자를 표현만 바꿔 반복 금지.
- **응답 버킷(코드북)은 가이드에서 만들지 않는다.** 측정 전에 보기를 정하면 질문이 그 보기에 맞춰 좁아진다 — 코드북은 인터뷰가 끝난 뒤 **실제 전사에서 귀납적으로** 만들어 결과의 '주제별' 탭에 낸다(스펙 C). 프로빙도 버킷에 매이지 않는다.
- **주제 카드 안에 질문 카드**가 들어가는 2단. 질문이 갖는 것(텍스트·알아낼 것·자극물)은 그대로.
- **헤더에 `최대 N턴` 실시간 표시**(N = 총질문수 + 주제수). 인터뷰 길이가 가이드 크기에 직결되므로 의뢰자가 편집하면서 즉시 보게 한다 — 엔진에 전체 상한이 없는 대신 여기서 보이게 한다.
- 순서 변경은 위/아래 버튼. 드래그 도입 안 함.

### 결과 (의뢰자) — 하위 탭 3개
> 정본: `docs/specs/2026-07-24-results-insight-report.md`
- `결과` 탭 아래 **요약 · 주제별 · 응답 N건** 3개. **상·하위 탭 모두 URL 로 표현**한다(`?tab=results&view=topics`) — 공유·뒤로가기·새로고침이 동작해야 한다.
- **요약 = 내러티브 리포트.** 문장이 주인공, 수치는 근거로 인라인. 발견 3~5개 = 헤드라인 + 근거 한 문장 + 대표 인용 1개. 그대로 의뢰자에게 보낼 수 있는 문서.
- **주제별** = 주제 > 질문 > 버킷·인용. 버킷 라벨은 `insight.codebooks`(전사에서 귀납 생성)에서, 없으면 구 가이드 버킷으로 폴백. 질문마다 `4명 중 2명이 이 질문을 받았습니다` 를 붙인다(턴 배분 때문에 질문별 모수가 다르다).
- **응답** = 세션별 전사. CSV·JSON 내보내기는 이 탭에 둔다.

#### 수치 표기 규칙 (정직성 — §0.1 계약 1의 연장)
- **`n < 10` 이면 `%` 를 쓰지 않는다.** `2 · 100%` 가 아니라 `2명 중 2명`. 분모를 항상 같이 적는다.
- **`n < 10` 이면 차트를 그리지 않는다.** 인라인 카운트로 낸다. `n >= 10` 부터 막대차트.
- **값 0 인 버킷은 `미언급 5개` 로 접는다.** 지우지 않는다 — 안 나온 것도 정보다.
- **유사 인용은 대표 1개.** 같은 취지가 반복되면 `4명이 같은 취지로 답했습니다` + 인용 1개.
- `n` = **그 질문을 실제로 받은 응답자 수**. 전체 응답 수가 아니다.

### 인터뷰 (응답자, 웜) — 데모 명확성 강화
- **구조는 현행 코드 그대로**(상단 진행자+진행률 → 질문 히어로 → 텍스트영역+마이크/다음 · 자극물 2분할). 색만 웜.
- **진행자 오브를 히어로로.** 헤더 구석 장식이 아니라 질문 위 중앙의 큰 오브(peach→red 라디얼). **실제 목소리 진폭으로 구동**한다 — `useTts().getLevel()`(이미 배선됨)을 rAF 로 읽어 오브 scale·글로우를 실시간 반응. 고정 맥동은 `getLevel` 없을 때·reduced-motion 폴백. 이모지 아님.
- **AI 상태를 한 줄로 명시**(오브 아래, `aria-live`): 생성 중 → **"생각 중이에요"** · TTS 재생 → **"답하는 중이에요"** · 녹음 → **"듣고 있어요"**. "AI가 지금 뭘 하는지"가 화면에 보이게.
- **NPU 속도를 보이게.** 발화 요청→응답 왕복을 재서 진행자 라벨 옆 텔레메트리 mono 칩(`RNGD · 0.3s`)으로. §2 mono=텔레메트리 전용. 서버 왕복 실측(상단 경과 타이머와 같은 정당성) — 벤치 값 아님을 구분.
- 질문이 주인공(display 급). 마이크=웜 red, 다음=red. `volume-2` 다시 듣기.
- 녹음 중: red 채움 맥동 + **실시간 입력 파형**(마이크 진폭, `transform:scaleY`만) + mono 타이머 `● 0:04` + `square` 정지. 전사 중: 스켈레톤 시머 + `RNGD · STT` 라벨.
- **응답자 여정 스테퍼**(동의→[자격]→인터뷰→완료): 상단 얇은 스텝 바로 "전체 중 어디인지". 완료=`check`·현재=red·미래=`ink-faint`. 부적격 종료 시 숨김.
