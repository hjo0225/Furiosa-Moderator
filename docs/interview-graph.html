인터뷰 한 판 = 잠들 줄 아는 실행 하나

완성형 그래프의 전체 구조. 매 턴 듣고(listen) → 고르고(strategize) → 만들고(generate) → 검수하고(guard) → 말한 뒤(speak), 응답자가 생각하는 동안 미리 일해두고(reflect) interrupt에서 잠든다. 발화가 오면 깨어나 다음 턴 — 세션이 끝날 때까지 이 원을 돈다.

그래프가 딛고 서는 것

노드들이 주고받는 공유 상태와, 그 상태를 매 턴 저장하는 체크포인터. 이 둘이 있어야 "잠들었다 깨는 실행"이 성립한다.

InterviewState — 그래프가 소유하는 상태 T2

대화·커버리지·페이스를 그래프 State로. 커버리지는 출석부(covered=[q1,q2])가 아니라 원장(ledger) — 문항별로 상태 · 알아낸 사실 · 안 판 떡밥 3칸을 기록한다. revisit과 정직한 종료의 근거.

PostgresSaver — 체크포인터 T1

thread_id = session_id. interrupt에서 잠들 때마다 상태 전체가 같은 Cloud SQL에 저장된다. 얻는 것: 일시정지·이어하기 · 크래시 복구 · 타임트래블("7턴에 왜 안 캐물었지?"를 그 시점 상태로 되감아 디버깅).

체크포인트 테이블재개 = Command(resume)재실행 규약: interrupt 앞 로직 멱등

이 그림이 티켓별로 차오르는 순서

T0

토대(그림 밖). chat(tools=) · embed() · langgraph 설치 · pgvector + Qwen3 자율 tool choice 실측 게이트

T1

뼈대. interrupt 루프 + 노드 5개(listen·strategize·generate·guard·speak) + PostgresSaver — 행동은 probe·advance·close 3종으로 시작

T2

상태. InterviewState + 커버리지 원장 — 정직한 종료와 revisit의 근거 확보

T3

행동 7종. clarify · challenge · revisit · redirect 추가 — 조건 엣지 분기 완성

T4

속도. reflect 슬로우패스(Send 병렬) + speak SSE 스트리밍 — 목표 체감 첫 토큰 1.5~2.5s

T5

도구. generate에 도구 4종 연결 + 브리핑 라이트 RAG(업로드 전용) — 코어와 독립, 병행 가능

LLM 콜 수는 기존 만능 1콜과 동일(패스트패스 기준) — 노드 분해는 콜 증가가 아니라 역할 분리다. Supervisor(자유 LLM 라우팅)는 쓰지 않는다: 구조화 출력 → 조건 엣지가 더 빠르고 디버깅 가능. 원본: LANGGRAPH_PLAN.md §4 · docs/TICKETS.md
